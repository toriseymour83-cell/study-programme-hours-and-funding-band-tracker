## Study Programme Hours and Funding Band Redesign

This folder shows how I would redesign the original study programme hours and funding band process using a more structured data workflow.

Instead of relying on an embedded spreadsheet inside individual learning agreements, the modern version separates the process into clear input, calculation, and reporting stages.

The timetable or session plan remains the starting point, but the data is stored in a structured format so that planned hours can be calculated consistently across all learners. Funding bands can then be assigned automatically using rules based on each learner’s calculated annual hours.

This approach makes the process easier to validate, easier to report on, and easier to scale across multiple students, employers, or cohorts.

The aim of this project is to demonstrate how the same operational problem can be handled using cleaner data structures, automated transformations, and management reporting.

### Workflow

1. Input timetable or session plan data
2. Store the data in a structured table
3. Calculate planned annual hours
4. Apply funding band rules
5. Produce validation checks and management reporting

### Example process

| Stage | Description |
|---|---|
| Input | Timetable or session plan data is entered in a structured format |
| Calculation | Planned hours are calculated consistently for each learner |
| Rules | Funding bands are assigned using predefined thresholds |
| Validation | Records can be checked for missing, low, or unusual hours |
| Reporting | Outputs can be used for management review and funding analysis |
